Installing MySQL 5.6 on Ubuntu

Mon, 13 May 2013

MySQL 5.6 has a lot of performance and replication improvements. Unfortunately MySQL 5.6 is not yet available through apt-get, so we will have to download the debian package and install it manually.

First of all we need to install libaio-dev

$ sudo apt-get install libaio-dev

Then install the MySQL package :

$ sudo dpkg -i mysql-5.6.11-debian6.0-x86_64.deb

Create MySQL user and group :

$ sudo groupadd mysql
$ sudo useradd -r -g mysql mysql

Now the bove package has installed mysql in /opt/mysql/server-5.6. We need to set the permissions right :

$ sudo chown -R root.root /opt/mysql
$ sudo chown -R mysql.mysql /opt/mysql/server-5.6

Now we should specify data directory :

$ sudo install -o mysql -g mysql -d /var/lib/mysql
$ sudo -u mysql /opt/mysql/server-5.6/scripts/mysql_install_db --user=mysql --datadir=/var/lib/mysql

Till now the system is not aware of Mysql, so we need to setup /etc/my.cnf

$ sudo vim /etc/my.cnf

And write these lined to it :

[mysqld]
basedir = /opt/mysql/server-5.6
datadir = /var/lib/mysql

To be able to make use of MySQL different scripts createa /etc/profile.d/mysql.sh file

$ sudo vim /etc/profile.d/mysql.sh

And paste the following into it :

PATH="/opt/mysql/server-5.6/bin:$PATH"
MANPATH="/opt/mysql/server-5.6/man:$MANPATH"

Then source it, or logout and log in again for the above changes to take effect :

$ source /etc/profile.d/mysql.sh

And we create the init.d scripts to start and stop the service, and set it to start on server startup :

$ sudo cp /opt/mysql/server-5.6/support-files/mysql.server /etc/init.d/mysql
$ sudo update-rc.d mysql defaults

You can now start the service for the first time :

$ sudo /etc/init.d/mysql start

One last step is to secure the MySQL installation because the root has no password so farand anyone can access it, run mysql_secure_installation and follow the dialogue :

$ sudo mysql_secure_installation
